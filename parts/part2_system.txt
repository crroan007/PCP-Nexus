    <h2><span class="section-number">03</span> What Is PCP Nexus?</h2>
    <p class="section-desc">The platform that automates your court document pipeline from inbox to network share.</p>
    <p>PCP Nexus is a workstation-based automation system that processes court document notifications from Tyler Technologies. It monitors an Outlook inbox around the clock, downloads PDF documents from Tyler's servers, renames or merges them according to court rules, and delivers the finished files to a shared network folder.</p>
    <h3>Core Capabilities</h3>
    <table>
      <thead><tr><th>Capability</th><th>What It Does</th></tr></thead>
      <tbody>
        <tr><td><strong>24/7 Autonomy</strong></td><td>Operates continuously without human intervention once started.</td></tr>
        <tr><td><strong>Legal Renaming</strong></td><td>Applies strict, court-compliant file naming rules automatically.</td></tr>
        <tr><td><strong>PDF Aggregation</strong></td><td>Joins related filings into a single PDF for Phase 2 petitions.</td></tr>
        <tr><td><strong>Deduplication</strong></td><td>Prevents duplicate filings from entering the pipeline. <span class="version-badge v44">v4.4</span></td></tr>
        <tr><td><strong>Verified Handoff</strong></td><td>SHA-256 verified file transfers prevent data loss. <span class="version-badge v44">v4.4</span></td></tr>
      </tbody>
    </table>

    <h3>The Two Pipelines</h3>
    <p>All documents flow through one of two pipelines, depending on the filing type:</p>
    <div style="display: grid; grid-template-columns: 1fr 1fr; gap: 16px; margin: 20px 0;">
      <div style="background: var(--white); border: 2px solid var(--blue-200); border-radius: var(--radius-lg); padding: 24px;">
        <span class="pipeline-badge phase1">&#9679; Phase 1</span>
        <h4 style="margin-top: 12px;">Civil Process / eAffidavits</h4>
        <p style="font-size: 14px; color: var(--gray-600); margin-bottom: 8px;">One email in &#8594; one renamed PDF out.</p>
        <p style="font-size: 13px; color: var(--gray-500);">Each incoming email produces exactly one PDF that is renamed according to court document type codes (e.g., AX02 &#8594; AX42 for Return of Service).</p>
      </div>
      <div style="background: var(--white); border: 2px solid var(--green-200); border-radius: var(--radius-lg); padding: 24px;">
        <span class="pipeline-badge phase2">&#9679; Phase 2</span>
        <h4 style="margin-top: 12px;">E-Filing / Petitions</h4>
        <p style="font-size: 14px; color: var(--gray-600); margin-bottom: 8px;">Multiple emails &#8594; one merged PDF.</p>
        <p style="font-size: 13px; color: var(--gray-500);">Related filings are collected and merged into a single PDF. The Lead Document (AX40) always comes first, followed by exhibits, then attachments.</p>
      </div>

PCP Nexus User Manual

Mission-critical document automation for court filings. This guide covers everything you need to operate, monitor, and maintain PCP Nexus — from daily checks to advanced configuration.

01 Quick Start

Get the system running in four steps.

  1. Open Outlook
    Launch Microsoft Outlook Classic and confirm it is logged into the correct mailbox. Outlook must be running as Administrator.
  2. Launch PCP Nexus
    Open the PCP Nexus application from the desktop or Start menu.
  3. Start the Engine
    Click Start Engine in the main window — or let Auto-Start handle it if enabled.
  4. Verify the Status Bar
    Watch the footer status bar. Once it turns green, the system is live and processing.
💡
Tip

All settings, databases, logs, and backups live in C:\ProgramData\PCP-Automation\. You should never need to navigate there during normal operations.

03 What Is PCP Nexus?

The platform that automates your court document pipeline from inbox to network share.

PCP Nexus is a workstation-based automation system that processes court document notifications from Tyler Technologies. It monitors an Outlook inbox around the clock, downloads PDF documents from Tyler's servers, renames or merges them according to court rules, and delivers the finished files to a shared network folder.

Core Capabilities

CapabilityWhat It Does
24/7 AutonomyOperates continuously without human intervention once started.
Legal RenamingApplies strict, court-compliant file naming rules automatically.
PDF AggregationJoins related filings into a single PDF for Phase 2 petitions.
DeduplicationPrevents duplicate filings from entering the pipeline. v4.4
Verified HandoffSHA-256 verified file transfers prevent data loss. v4.4

System Architecture

PCP Nexus uses four specialized software agents that work together in sequence:

The Two Pipelines

All documents flow through one of two pipelines, depending on the filing type:

● Phase 1

Civil Process / eAffidavits

One email in → one renamed PDF out.

Each incoming email produces exactly one PDF that is renamed according to court document type codes (e.g., AX02 → AX42 for Return of Service).

● Phase 2

E-Filing / Petitions

Multiple emails → one merged PDF.

Related filings are collected and merged into a single PDF. The Lead Document (AX40) always comes first, followed by exhibits, then attachments.

09 Phase 1 Workflow: Affidavits

● Phase 1  One email in, one renamed PDF out.

Each Tyler notification email for a civil process document is handled individually. The system downloads the PDF, applies the appropriate document-type rename code, and delivers the file to the network share.

Document Rename Map

Phase 1 applies court-compliant rename codes automatically:

Source CodeOutput CodeDocument Type
AX02AX42Return of Service
AX07AX47Attorney Motion

Additional rename mappings follow the same pattern. Case numbers are extracted exactly as displayed in the source email.

10 Phase 2 Workflow: Petitions

● Phase 2  Multiple documents merge into one PDF.

Petition filings arrive as multiple separate emails. PCP Nexus collects all related documents and merges them into a single PDF in a strict order.

Merge Order

  1. AX40 — Lead Document
    Always placed first in the merged PDF. Nothing merges until the Lead Document arrives.
  2. Exhibits
    Placed in the middle of the merged document, in the order received.
  3. Attachments
    Placed at the end of the merged PDF.

Excluded Document Types

The following are automatically filtered out and will not appear in the merged PDF:

CodeReason
AX69Excluded by court rules
AX81Excluded by court rules
CL + digitsClerk Letters — excluded from merge v4.4
🆕
v4.4 Fix — Clerk Letter Detection

Filenames containing "CL" followed by digits (e.g., AX40...CL2.pdf) are now correctly identified as Clerk Letters and excluded from the merge. Earlier versions could miss this pattern.

💡
Wait State

If exhibits or attachments arrive before the Lead Document (AX40), they are held in a queue. Merging only begins once the Lead Document is present.

12 Lead Document Deduplication v4.4

How PCP Nexus prevents duplicate petition filings.

When a petition is amended and filed more than once, the system automatically compares the Date Submitted values. Only the newest submission is kept — the older version is discarded. This ensures only one row per unique lead document appears in the CSV and only the most current filing enters the merged PDF.

Fig. 4 — Deduplication: when duplicate lead documents arrive, only the newest is retained

13 Safe CSV Handoff v4.5

How your daily CSV report is securely transferred every night.

Each night, PCP Nexus transfers the day's in-progress CSV to its final archive destination using a verified, four-step process:

  1. Copy
    The in-progress CSV is copied to the archive destination. v4.5 This destination is now the root PDF output folder, bypassing the legacy DONE folder.
  2. Verify
    A SHA-256 hash confirms the copy is completely identical to the source file. If the hashes don't match, the handoff is aborted and retried.
  3. Delete
    The in-progress copy is deleted only after verification passes.
  4. Roll Forward
    A blank CSV with tomorrow's date is created, ensuring late-arriving emails don't mix with today's completed data.
🆕
New in v4.5 — Independent Handoff Polling

The CSV handoff routine now runs on its own 5-minute polling cycle, completely independent of the main engine state. This means the handoff will execute on schedule even if the engine is stopped or idle.

Fig. 5 — Safe CSV handoff: copy → verify → delete → roll forward

14 Hourly CSV-to-Database Backfill v4.4

Automatic gap prevention for the internal database.

Every 60 minutes, PCP Nexus parses the current in-progress CSV and inserts any missing records into the internal SQLite database. This prevents tracking gaps that could occur if the application crashed mid-cycle or was restarted unexpectedly.

💡
This Runs Automatically

No operator action is required. The backfill process is silent and runs in the background. You can verify it's working by checking the database record counts against the CSV row counts during weekly spot-checks.

15 Keyword Monitoring & Exceptions

How the system flags documents that need human review.

PCP Nexus scans the Accepted Comments field of each email for specific legal keywords. When a match is found, the document processes normally but is also flagged for manual review.

What Happens When a Keyword Is Detected

  1. Document Processes Normally
    The PDF is renamed/merged and delivered as usual — processing is not blocked.
  2. Email Routes to Exceptions
    The source email is moved to the Exceptions folder for visibility.
  3. Dashboard Alert Appears
    A red alert indicator appears on the Dashboard in the Manual Review Queue.

Monitored Keywords

deny fee dismissed dwop
⚠️
Always Review Keyword Hits

These keywords may indicate legally significant changes (denied filings, dismissed cases, fee requirements). Review the full comment text in the Dashboard's "Comments & Follow-up" section before clearing the alert.
